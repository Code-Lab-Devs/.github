# Contributing to Code Lab Projects

Welcome to the Code Lab contribution guide! Since we are a learning community, we follow a standard professional workflow to ensure our code is clean and everyone gets experience with Git & GitHub.

## 🚫 The Golden Rule
**Never push directly to the `main` branch.** 
All changes must be made on a separate branch and submitted via a **Pull Request (PR)**. This allows team members to review your code, catch bugs, and learn from each other.

## 🛠️ Step-by-Step Contribution Guide

### 1. Create a Branch
Before writing any code, create a new branch from `main`. Name it logically based on what you are doing:
*   `feat/add-login-page` (for new features)
*   `fix/navbar-bug` (for bug fixes)
*   `docs/update-readme` (for documentation)

### 2. Write Clear Commit Messages
When saving your work, use clear messages so your team knows what you did. We recommend the following format:
*   `feat: added user authentication`
*   `fix: resolved crash on startup`
*   `style: formatted CSS for mobile view`

### 3. Open a Pull Request (PR)
Once your code is ready:
1. Push your branch to GitHub.
2. Open a **Pull Request** targeting the `main` branch.
3. Give your PR a clear title and a description of what you changed.
4. Tag a teammate or an Admin to review your code.

### 4. Code Review
*   **If you are reviewing:** Be kind, constructive, and explain *why* you are suggesting a change. Remember, we are all learning!
*   **If you are receiving feedback:** Don't take it personally. Code reviews are about improving the code, not criticizing the coder.

## 🏆 Challenge Repositories
If you are participating in a **Code Lab Team Challenge**, please follow the specific rules given by the Admins. Usually, only your specific team will have "Write" access to the repo, and you will collaborate exclusively with your team members.

Thank you for helping build the IT community at the University of Tripoli!
